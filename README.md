# LangGraph POC — ADF / ADLS / ServiceNow with LangSmith Tracing

A LangGraph multi-agent proof-of-concept that analyses Azure Data Factory (ADF) pipeline runs
and exposes the agent as both a REST API and an MCP (Model Context Protocol) server.

---

## Project Structure

```text
LG-test/
├── main.py          # Simple LangGraph hello agent with LangSmith tracing
├── simple.py        # Minimal LangGraph hello agent (no tracing)
├── adf_agents.py    # LangGraph ADF analysis agent (LLM-powered)
├── adf_tools.py     # Azure Data Factory SDK client + standalone MCP server
├── api.py           # FastAPI server exposing REST endpoints and MCP endpoint
├── mcp_client.py    # MCP test client (connects to api.py over StreamableHTTP)
├── pyproject.toml   # Project dependencies (managed with uv)
└── .env.example     # Environment variable template — copy to .env and fill in
```

---

## File Descriptions

### `main.py`

A minimal LangGraph graph with a single `hello_agent` node. Its main purpose is to
verify that LangSmith tracing is wired up correctly. Reads `LANGCHAIN_*` env vars and
prints whether tracing is enabled, then invokes the graph with a "Hello World" message.

### `simple.py`

Same hello-agent graph as `main.py` but with no tracing or environment variable setup.
Useful as a bare-minimum LangGraph reference or smoke test.

### `adf_agents.py`

A LangGraph agent that uses an LLM (Azure OpenAI or standard OpenAI) to analyse ADF
pipeline runs. Graph topology:

```text
START → agent ──── has tool_calls? ──→ tools → agent
              └─── no ──────────────→ END
```

The agent calls `get_pipeline_hierarchy` (from `adf_tools.py`) to retrieve the full
pipeline execution tree, then produces a structured diagnosis with failures, errors, and
performance notes. Can be run as a CLI or imported programmatically.

### `adf_tools.py`

Contains two ADF client classes and a standalone MCP server:

- **`ADFClient`** — synchronous wrapper around the Azure Data Factory Management SDK.
  Fetches pipeline runs, activity runs, and builds the full parent/child hierarchy using
  a thread pool for parallel child lookups.
- **`AsyncADFClient`** — async version of the above; wraps the sync SDK with
  `asyncio.run_in_executor` so it integrates cleanly with FastAPI and LangGraph.
- **MCP server** — exposes `get_pipeline_hierarchy` as an MCP tool via StreamableHTTP.
  Can run standalone on port 8001 (`uv run uvicorn adf_tools:app --port 8001`).

Authentication uses `DefaultAzureCredential` (Azure CLI login, managed identity, etc.).

### `api.py`

The main server. Combines a FastAPI REST API with a FastMCP MCP server mounted on the
same process:

| Endpoint  | Method | Description                                 |
| --------- | ------ | ------------------------------------------- |
| `/`       | GET    | API info and available endpoints            |
| `/health` | GET    | Health check                                |
| `/invoke` | POST   | Run the LangGraph hello agent over REST     |
| `/mcp`    | POST   | MCP StreamableHTTP endpoint for MCP clients |
| `/docs`   | GET    | Swagger UI (auto-generated by FastAPI)      |

MCP tools exposed: `hello_agent` and `analyse_adf_pipeline`.

### `mcp_client.py`

A test client that connects to the MCP server in `api.py` via StreamableHTTP. It:

1. Initialises a session with the server
2. Lists all available tools
3. Calls `hello_agent` with a test message
4. Calls `analyse_adf_pipeline` if `ADF_RUN_ID` is set in `.env`

## Setup

### 1. Install dependencies

The project uses [uv](https://docs.astral.sh/uv/). If you don't have it:

```bash
pip install uv
```

Sync dependencies (including Azure SDK and OpenAI extras):

```bash
uv sync --extra adf --extra agent
```

> The `.venv` directory is already present if you cloned this repo with its environment.

### 2. Configure environment variables

```bash
cp .env.example .env
```

Edit `.env` and fill in the required values:

```env
# LangSmith tracing (optional)
LANGCHAIN_TRACING_V2=true
LANGCHAIN_API_KEY=lsv2_your_api_key_here
LANGCHAIN_PROJECT=langgraph-poc

# Azure Data Factory (required for ADF features)
AZURE_SUBSCRIPTION_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
AZURE_RESOURCE_GROUP=my-resource-group
AZURE_DATA_FACTORY_NAME=my-adf-name

# ADF pipeline run ID to analyse
ADF_RUN_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx

# LLM — Option A: Azure OpenAI
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com/
AZURE_OPENAI_API_KEY=<your-azure-openai-key>
OPENAI_MODEL=gpt-4o
OPENAI_API_VERSION=2024-08-01-preview

# LLM — Option B: Standard OpenAI
# OPENAI_API_KEY=sk-...
# OPENAI_MODEL=gpt-4o-mini
```

## How to Run

### Option A — Hello agent (LangSmith tracing test)

```bash
uv run python main.py
```

Runs a simple LangGraph graph, prints the result, and logs a trace to LangSmith if
`LANGCHAIN_TRACING_V2=true`.

---

### Option B — Minimal hello agent (no tracing)

```bash
uv run python simple.py
```

---

### Option C — ADF analysis agent (CLI)

```bash
uv run python adf_agents.py
```

Requires `ADF_RUN_ID` in `.env` and Azure credentials. Analyses the pipeline run and
prints a structured diagnosis to the terminal.

---

### Option D — Full server + MCP client (main workflow)

**Terminal 1 — Authenticate and start the server:**

```bash
# Authenticate Azure CLI so DefaultAzureCredential can reach ADF
az login

cd c:\Users\submedip\Downloads\LG-test
uv run uvicorn api:app --reload --port 8001
```

Expected output:

```text
INFO  [ADF Async] Client initialized successfully
INFO:     Uvicorn running on http://0.0.0.0:8001
```

> If you see `Client not initialized (missing config or SDK)`, check that
> `AZURE_SUBSCRIPTION_ID`, `AZURE_RESOURCE_GROUP`, and `AZURE_DATA_FACTORY_NAME`
> are all set in `.env`.

**Terminal 2 — Run the MCP test client:**

```bash
cd c:\Users\submedip\Downloads\LG-test
uv run python mcp_client.py
```

The client connects to `http://localhost:8001/mcp`, lists tools, calls `hello_agent`,
and (if `ADF_RUN_ID` is set) calls `analyse_adf_pipeline` with the configured run ID.

---

### Option E — Standalone ADF MCP server

If you only need the ADF tools without the hello agent:

```bash
uv run uvicorn adf_tools:app --reload --port 8001
```
